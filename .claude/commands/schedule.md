## 작업 순서

1. **사용자에게 기간 확인**
   - 어느 기간의 작업을 일정으로 등록할지 사용자에게 물어보기
   - 예: "어느 기간의 작업을 일정으로 등록할까요? (시작일-종료일)"
   - 기본값: 최근 2주간

2. **DAILY SCRUM 데이터 수집**
   - Database ID: `2fc26170-466f-80e2-999f-cf5ac1963e2f`
   - 지정된 기간의 모든 DAILY SCRUM 페이지 조회
   - 각 페이지의 "금일 예정 업무 보고" 테이블 내용 읽기

3. **작업 분석 및 그룹화**
   - 수집한 작업들을 유사한 기능별로 그룹화 (5-10개 정도)
   - 각 그룹에 대해:
     - 적절한 기능명 부여 (예: "인증 기능 구현", "상품 목록 시스템" 등)
     - 해당 기능의 시작일과 종료일 파악
     - 포함된 세부 작업 목록 정리

4. **Schedule List에 일정 등록**
   - Database ID: `2fc26170-466f-8131-8a3c-d433dcda1cd4`
   - 각 기능별 그룹에 대해 페이지 생성:
     - Name (title): 기능명
     - 일정 (date): `{"start": "YYYY-MM-DD", "end": "YYYY-MM-DD"}`
     - 참여자 (people): 강주현 (ID: `7c32774b-0096-4545-a9fe-7cfec90faa15`)

5. **일정 페이지에 세부 내용 추가**
   - 각 일정 페이지에 해당 기능의 세부 작업들을 bulleted list로 추가
   - 예:
     ```
     - 로그인/회원가입 페이지 컴포넌트 구조화
     - 회원가입 폼 필드 구현

     ```
